# CoWork + Claude in Chrome Reliability Report
**Date:** 2026-03-04
**Status:** Baseline assessment

---

## Reliability Signal: **KNOWN ISSUES—NOT BROADLY BROKEN**

The integration is **not universally unreliable**, but specific failure modes are well-documented and consistently reported across GitHub issues, support threads, and expert commentary. Dan Guido (trail of bits) published what he describes as "the internet's most authoritative troubleshooting thread" for the integration, signaling that problems exist but are solvable with the right diagnosis.

**Community sentiment:** The integration works in isolation, but coexistence with other Claude desktop tools creates a "silently broken" state—everything looks correct, but the connection fails.

---

## Failure Modes Observed

### 1. **CoWork ↔ Claude Code Mutual Exclusivity** (Critical)
- **Symptom:** tabs_context_mcp tool returns "Browser extension is not connected" despite correct installation and configuration.
- **Root cause:** Both Claude Desktop (Cowork) and Claude Code CLI register native messaging host configs for the same Chrome extension, but use different binaries and incompatible socket formats.
- **Why it's hard to debug:** Everything appears working—extension is installed, native host is running, socket exists.
- **Impact:** Using Claude Desktop's Cowork feature silently breaks Claude Code's Chrome integration. The reverse is also true.
- **Known workaround:** Disable the native messaging config for whichever tool you're NOT using. Only enable the Claude extension in ONE Chrome profile.
- **Source:** [Issue #29057](https://github.com/anthropics/claude-code/issues/29057); [Dan Guido on X](https://x.com/dguido/status/2015675500799173013)

### 2. **Connection Drops During Browser Automation** (Moderate)
- **Symptom:** Chrome fully quits or extension disconnects after 2–3 minutes of browser automation via Claude in Chrome on some macOS systems. Extension "permanently disconnects" after screenshot/scroll operations.
- **Root cause:** Chromium DevTools Protocol (CDP) session detaches during slow operations (screenshots ~20s), with no graceful recovery mechanism.
- **Workaround:** Restart Chrome or the extension.
- **Source:** [Issue #28013](https://github.com/anthropics/claude-code/issues/28013)

### 3. **MCP Server Connection Failure After Desktop Update** (Moderate)
- **Symptom:** After updating Claude Desktop to the latest version, Cowork cannot connect to the mcp-registry or Claude in Chrome MCP servers. Error banners appear: "Connection could not be established."
- **Root cause:** The native host wrapper at `~/.claude/chrome/chrome-native-host` may have a hardcoded version that becomes stale after updates.
- **Source:** [Issue #27625](https://github.com/anthropics/claude-code/issues/27625)

### 4. **Multi-Profile Conflicts** (Moderate)
- **Symptom:** If the Claude extension is installed in multiple Chrome profiles, each spawns its own native host and socket, leading to connection issues.
- **Workaround:** Install the extension in a single profile only.
- **Source:** Multiple issues across the tracker

### 5. **Initial Connection Failures** (Minor)
- **Symptom:** Extension fails to connect on first launch despite correct setup.
- **Fix:** Restart the extension or update Claude Desktop and the extension to the latest versions.
- **Source:** [Issue #29528](https://github.com/anthropics/claude-code/issues/29528); official troubleshooting docs

### 6. **Performance Degradation**
- **Symptom:** Chrome automation runs noticeably slower than manual interaction for repetitive tasks.
- **Known limitation:** This is acknowledged behavior, not a bug. Expect extended timelines for browser-automated workflows.
- **Source:** CoWork documentation and multiple user reports

## Recommended Actions (Short Term)

1. **Audit active tools:** Determine if both Cowork and Claude Code are in use. If only Cowork is needed, there's no interaction risk.

2. **If using both Cowork and Claude Code:**
   - Use separate Chrome profiles: one for Cowork, one for Claude Code
   - OR disable the native messaging host for Claude Code if Cowork is primary
   - Edit `~/.claude/claude-code-native-host.json` to verify it's pointing to the correct binary, not Cowork's

3. **If connection drops during automation:**
   - Restart the Chrome extension immediately (unload/reload)
   - Consider breaking long automation tasks into shorter, sequential runs
   - Monitor console for error patterns

4. **If MCP servers won't connect after a Claude Desktop update:**
   - Delete `~/.claude/chrome/chrome-native-host` (wrapper will rebuild)
   - Restart Claude Desktop
   - Re-enable Claude in Chrome in Connector settings

5. **Verify single-profile setup:** Confirm Claude in Chrome is installed in exactly one Chrome profile, not multiple.

## Trend & Baseline

**First baseline assessment.** Issues are chronic (reported across 2024–2026) but well-understood by the community and engineering team. No signs of regression or new systemic failures in March 2026. The integration appears stable in isolation and when used correctly (single tool, single profile, latest versions).
