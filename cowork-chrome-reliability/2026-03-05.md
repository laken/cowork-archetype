**Date:** 2026-03-05
**Compared to:** 2026-03-04 (baseline)

## Reliability Signal: **BROADLY REPORTED — NO NEW FIXES**

Community sentiment is unchanged from yesterday: the integration is unreliable in specific, well-documented scenarios, and the open bug queue on GitHub has grown. The most prominent new signal is that issue #29933 (MCP connection failure on Apple Silicon M4 Pro) was **closed today as a duplicate** of #29528 — signaling Anthropic recognizes this as a known, tracked issue, but no fix has shipped. No evidence of broad success in the last 24 hours that would suggest this is isolated.

---

## Failure Modes Observed (Last 24–48 Hours)

### 1. **Apple Silicon macOS Sequoia — Complete Connection Failure** (Critical, Ongoing)
- **Issue:** Extension returns "No Chrome extension connected" persistently on M4 Pro (and M2 Pro variants). Extension appears installed; connection never establishes.
- **Status:** Issue #29933 closed as duplicate of #29528. No fix shipped.
- **Affected config:** macOS Sequoia, Apple Silicon M2/M4, extension v1.0.56.
- **Source:** [Issue #29933](https://github.com/anthropics/claude-code/issues/29933), [Issue #29528](https://github.com/anthropics/claude-code/issues/29528)

### 2. **CoWork ↔ Claude Code Mutual Exclusivity** (Critical, Ongoing)
- **Issue:** Claude Desktop Cowork and Claude Code CLI both register native messaging hosts for the same extension ID. Chrome picks one; the other silently fails. Everything looks correct — extension installed, host running, socket exists — but connection never happens.
- **Status:** Issue #29057 remains **open and unresolved** as of March 5. No assigned developer, no linked PRs.
- **Workaround:** Rename the native messaging `.json` for whichever tool is NOT in use to `.disabled` in `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/`, then restart Chrome.
- **Source:** [Issue #29057](https://github.com/anthropics/claude-code/issues/29057), [Dan Guido on X](https://x.com/dguido/status/2015675500799173013)

### 3. **Silent Disconnect During Session** (Moderate, Ongoing)
- **Issue:** Extension stops responding mid-session after 10–30 minutes with no error or warning. Requires Chrome restart to recover.
- **Status:** Persistent across multiple issue reports; no new resolution noted.
- **Source:** [Issue #26449](https://github.com/anthropics/claude-code/issues/26449), [Issue #28013](https://github.com/anthropics/claude-code/issues/28013)

### 4. **Permanent Disconnect After Screenshot/Scroll** (Moderate, Ongoing)
- **Issue:** Extension enters unrecoverable disconnected state following screenshot or scroll operations. Full Chrome restart required.
- **Source:** [Issue #28013](https://github.com/anthropics/claude-code/issues/28013)

### 5. **Multi-Profile Conflict** (Moderate, Ongoing)
- **Issue:** Extension installed in multiple Chrome profiles causes socket conflicts; only one profile can connect at a time.
- **Workaround:** Single Chrome profile only.
