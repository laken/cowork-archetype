## Failure Modes Observed

Same set as yesterday. No new failure modes identified in the last 24 hours.

1. **"No Chrome extension connected" on startup** — Persists on macOS Sequoia + Apple Silicon. ([#29933](https://github.com/anthropics/claude-code/issues/29933), [#25151](https://github.com/anthropics/claude-code/issues/25151))

2. **Mid-session disconnects after screenshot/scroll** — CDP session detaches, enters unrecoverable exponential backoff. Requires full Chrome restart. ([#28013](https://github.com/anthropics/claude-code/issues/28013), [#26449](https://github.com/anthropics/claude-code/issues/26449))

3. **CoWork ↔ Claude Code CLI conflict** — Competing native messaging hosts, incompatible socket formats. Only one can work at a time; the other silently fails. ([#29057](https://github.com/anthropics/claude-code/issues/29057))

4. **MCP registry regression (post-Feb 13-14 auto-update to Desktop v1.1.3189)** — Red error banners on mcp-registry + Claude in Chrome at every session start. ([#27492](https://github.com/anthropics/claude-code/issues/27492)) — **Still open, day 26, no patch.**

5. **Chrome Canary hijack** — Browser tools route to Canary even without extension installed there. ([#31771](https://github.com/anthropics/claude-code/issues/31771))

6. **Auth/session mismatch** — Stale session tokens cause MCP bridge rejection. Re-auth doesn't fix. ([#24593](https://github.com/anthropics/claude-code/issues/24593))

7. **Service worker goes idle during extended sessions** — Acknowledged by Anthropic in docs. Mitigation: run `/chrome → Reconnect extension`.

8. **`chromeEnabled: true` in settings.json doesn't reliably connect** — `claude --chrome` from CLI works; the settings flag does not always trigger connection. ([#31802](https://github.com/anthropics/claude-code/issues/31802), filed March 7, open.)

## Trend

**Flat to marginally better on issue closure count, but not on the core regression.**

Two recent issues were closed (#32496 on March 9, #29938 on March 2), but neither appears to represent a systematic fix for the native messaging host conflict or the MCP registry regression. The Feb 13-14 regression is now **day 26** with no public patch. No new app or extension releases today.

**Bottom line:** Same advice as prior days. (1) Don't run CoWork and Claude Code CLI simultaneously — manually disable the conflicting native messaging host. (2) Remove Chrome Canary if installed. (3) Use `/chrome → Reconnect extension` if tools stop responding mid-session. (4) Full Chrome restart for unrecoverable backoff loops. (5) Use `claude --chrome` from Terminal rather than relying on `chromeEnabled: true` in settings.
