## Failure Modes Observed

No new failure modes identified. The canonical list is unchanged from [yesterday's briefing](2026-03-12.md):

1. **"No Chrome extension connected" on startup** — macOS Sequoia + Apple Silicon. ([#29933](https://github.com/anthropics/claude-code/issues/29933), [#25151](https://github.com/anthropics/claude-code/issues/25151), [#31971](https://github.com/anthropics/claude-code/issues/31971))
2. **Mid-session disconnects after screenshot/scroll** — CDP session detaches, enters unrecoverable exponential backoff. Requires full Chrome restart. ([#28013](https://github.com/anthropics/claude-code/issues/28013), [#26449](https://github.com/anthropics/claude-code/issues/26449))
3. **CoWork ↔ Claude Code CLI conflict** — Both register native messaging hosts for the same extension ID. Only one works at a time. ([#29057](https://github.com/anthropics/claude-code/issues/29057), [#33483](https://github.com/anthropics/claude-code/issues/33483) — new duplicate filed March 12)
4. **MCP registry regression (post-Feb 13-14 auto-update to Desktop v1.1.3189)** — ([#27492](https://github.com/anthropics/claude-code/issues/27492)) — **Still open, day 28, no patch.**
5. **Chrome Canary hijack** — Browser tools route to Canary even without extension installed. ([#31771](https://github.com/anthropics/claude-code/issues/31771))
6. **Auth/session mismatch** — Stale session tokens cause MCP bridge rejection. ([#24593](https://github.com/anthropics/claude-code/issues/24593))
7. **Service worker idle disconnect** — Acknowledged by Anthropic in docs. Mitigation: `/chrome → Reconnect extension`.
8. **`chromeEnabled: true` in settings.json unreliable** — `claude --chrome` from CLI works; the settings flag does not. ([#31802](https://github.com/anthropics/claude-code/issues/31802))
9. **MCP shows connected, extension reports not connected** — Inconsistent state. ([#32247](https://github.com/anthropics/claude-code/issues/32247))
