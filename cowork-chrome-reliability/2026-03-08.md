# CoWork + Claude in Chrome Reliability Briefing — 2026-03-08

## Reliability Signal

**Broadly reported, not isolated.** Connection failures between CoWork/Claude Desktop and the Claude in Chrome extension on macOS remain a well-documented, multi-issue problem. GitHub issues span from early February through early March 2026 with no resolution announced. The Chrome Web Store rating sits at 3.4/5 stars, and reviewers consistently flag reliability as the weak point. Dan Guido (Trail of Bits CEO) has been vocal on X about the debugging difficulty, calling the Cowork/Code conflict "infuriating." Community sentiment: the feature is promising but production-unreliable for daily use.

## Failure Modes Observed

1. **Native messaging host conflict (Cowork vs. Claude Code):** Both register configs for the same Chrome extension using incompatible socket formats. Cowork silently breaks Code's Chrome integration and vice versa. Everything appears correct — extension installed, native host running, socket exists — but connection never establishes. ([GitHub #29057](https://github.com/anthropics/claude-code/issues/29057), [Dan Guido on X](https://x.com/dguido/status/2015675500799173013))

2. **macOS Sequoia + Apple Silicon M4 Pro total failure:** MCP connection consistently fails with "No Chrome extension connected" on M4 Pro machines. Completely unusable. ([GitHub #29933](https://github.com/anthropics/claude-code/issues/29933))

3. **Mid-session silent disconnects:** Extension drops connection without warning during screenshot/scroll operations, enters infinite exponential backoff reconnection loop (100+ attempts over ~2.5 hours), never recovers. Requires full Chrome restart. ([GitHub #28013](https://github.com/anthropics/claude-code/issues/28013), [GitHub #26449](https://github.com/anthropics/claude-code/issues/26449))

4. **MCP registry regression after v1.1.3189 update:** Cowork stopped connecting to mcp-registry and Claude in Chrome MCP servers after the ~Feb 13-14 auto-update. Persists on latest builds. ([GitHub #27492](https://github.com/anthropics/claude-code/issues/27492))

5. **Chrome Canary interference:** Cowork browser tools connect to Chrome Canary instead of regular Chrome when both are installed, even if the extension exists only in regular Chrome. ([GitHub #31771](https://github.com/anthropics/claude-code/issues/31771))

6. **Auth/session handshake failures:** MCP bridge rejects connection requests due to identity mismatch; re-authentication doesn't resolve. Suggests stale session tokens or cached identity state. ([GitHub #24593](https://github.com/anthropics/claude-code/issues/24593))

**Bottom line:** If you're using Cowork with Claude in Chrome on macOS today, expect connection failures. The native messaging host conflict between Cowork and Claude Code is the most common root cause. The workaround is disabling whichever native host config you're not actively using and restarting Chrome.
