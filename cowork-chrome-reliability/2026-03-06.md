# CoWork + Claude in Chrome — Daily Reliability Briefing
**Date:** 2026-03-06

---

## Reliability Signal

**Broadly reported, not isolated.** The GitHub issue tracker for `anthropics/claude-code` has 10+ open issues specifically about Chrome extension connection failures on macOS, dating from late 2025 through March 2026. Dan Guido (Trail of Bits CEO) called out the problem publicly on X, describing it as "infuriating to debug." The official troubleshooting page was updated today, and the feature remains labeled **beta**. Community sentiment: frustrated but engaged — people want this to work and are filing detailed bug reports.

## Failure Modes Observed

1. **Native messaging host conflict (root cause for many):** Claude Code CLI and Claude Desktop (CoWork) both register native messaging host configs for the same Chrome extension using different binaries and incompatible socket formats. Installing both silently breaks whichever one you're not actively using. ([#29057](https://github.com/anthropics/claude-code/issues/29057), [Dan Guido on X](https://x.com/dguido/status/2015675500799173013))

2. **"No Chrome extension connected" on Apple Silicon:** Persistent failure on macOS Sequoia + M4 Pro. Extension toggling, Chrome restarts, reinstalls — none resolve it. ([#29933](https://github.com/anthropics/claude-code/issues/29933))

3. **MCP registry connection failure after auto-update:** CoWork lost ability to connect to mcp-registry and Claude in Chrome MCP servers after Claude Desktop auto-updated to v1.1.3189 (~Feb 13–14). Browser automation entirely non-functional. ([#27492](https://github.com/anthropics/claude-code/issues/27492))

4. **Mid-session disconnects:** Extension silently disconnects during active use, especially during screenshot/scroll operations. Enters unrecoverable state requiring full Chrome restart. ([#26449](https://github.com/anthropics/claude-code/issues/26449), [#28013](https://github.com/anthropics/claude-code/issues/28013))

5. **Startup race condition:** If Chrome + extension wasn't ready when Claude Code/Desktop started, connection fails for the entire session. No automatic retry. ([#29528](https://github.com/anthropics/claude-code/issues/29528))

6. **Multiple Chrome profiles:** Extensions installed across multiple profiles spawn competing native hosts and sockets, causing conflicts.

7. **Edge browser panel closure:** Side panel closes on tab switch in Microsoft Edge on macOS. Chrome handles this correctly. ([#30873](https://github.com/anthropics/claude-code/issues/30873))

## Trend (vs. March 5)

**Flat — no improvement.** Yesterday's report noted #29933 was closed as a duplicate of #29528, confirming Apple Silicon connection failures as a tracked bug class. Today: no new fixes shipped, no new extension version, #29057 (CoWork/Claude Code mutual exclusivity) remains open and unassigned. The official troubleshooting page was updated today but with no substantive new guidance — still generic steps (refresh, disable conflicting extensions, clear cache).

The Claude in Chrome release notes page returned a 404 on direct fetch today, which may indicate a URL restructure or temporary issue. No new Claude Desktop/CoWork patch detected. The issue backlog on GitHub remains large and growing.

**Bottom line:** Third consecutive day with no fix shipping for any of the core failure modes. Community workarounds (rename native messaging host configs, single Chrome profile, correct startup order) remain the only path to reliability. The feature is still beta-labeled. No signal that a targeted fix for the native host conflict or mid-session disconnects is imminent.
