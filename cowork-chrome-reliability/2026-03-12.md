# CoWork + Claude in Chrome Reliability Briefing — 2026-03-12

**Broadly reported, still unresolved.** No new wave of reports in the last 24 hours. The same core GitHub issues remain open. [Issue #31971](https://github.com/anthropics/claude-code/issues/31971) (filed March 8, "Browser extension is not connected" on macOS Sequoia + Chrome 145) was labeled as a duplicate, confirming Anthropic is aware of the underlying connection problem but hasn't shipped a fix.

Dan Guido's X thread ([link](https://x.com/dguido/status/2015675676766916761)) remains the community's go-to troubleshooting reference and continues to circulate. Community sentiment: **unchanged — frustrated, relying on manual workarounds.**

## Failure Modes Observed

No new failure modes identified today. The canonical list remains:

1. **"No Chrome extension connected" on startup** — macOS Sequoia + Apple Silicon. ([#29933](https://github.com/anthropics/claude-code/issues/29933), [#25151](https://github.com/anthropics/claude-code/issues/25151), [#31971](https://github.com/anthropics/claude-code/issues/31971))

2. **Mid-session disconnects after screenshot/scroll** — CDP session detaches, enters unrecoverable exponential backoff (100+ reconnect attempts over ~2.5 hours). Requires full Chrome restart. ([#28013](https://github.com/anthropics/claude-code/issues/28013), [#26449](https://github.com/anthropics/claude-code/issues/26449))

3. **CoWork ↔ Claude Code CLI conflict** — Both register native messaging hosts for the same extension ID using different binaries and incompatible socket locations. Only one can work at a time. ([#29057](https://github.com/anthropics/claude-code/issues/29057))

4. **MCP registry regression (post-Feb 13-14 auto-update to Desktop v1.1.3189)** — Red error banners on mcp-registry + Claude in Chrome at every session start. ([#27492](https://github.com/anthropics/claude-code/issues/27492)) — **Still open, day 27, no patch.**

5. **Chrome Canary hijack** — Browser tools route to Canary even without extension installed there. ([#31771](https://github.com/anthropics/claude-code/issues/31771))

6. **Auth/session mismatch** — Stale session tokens cause MCP bridge rejection. Re-auth doesn't fix. ([#24593](https://github.com/anthropics/claude-code/issues/24593))

7. **Service worker idle disconnect** — Acknowledged by Anthropic in docs. Mitigation: `/chrome → Reconnect extension`.

8. **`chromeEnabled: true` in settings.json unreliable** — `claude --chrome` from CLI works; the settings flag does not always trigger connection. ([#31802](https://github.com/anthropics/claude-code/issues/31802))

9. **MCP shows connected, extension reports not connected** — Inconsistent state between MCP server and extension status. ([#32247](https://github.com/anthropics/claude-code/issues/32247))

## Trend

**Flat. Day 27 of the February regression with no public patch.**

The March 8 duplicate closure (#31971) confirms awareness but not resolution. No new app or extension releases today. No Reddit signal found — discussion remains concentrated on GitHub issues and X.

The mid-February auto-update regression (#27492) is approaching one month open without a targeted fix. The native messaging host conflict between CoWork and Claude Code CLI (#29057) has been understood and documented by the community (particularly by Dan Guido) but there's been no architectural resolution from Anthropic.

**Bottom line:** Same advice as prior days. (1) Don't run CoWork and Claude Code CLI simultaneously — disable the conflicting native messaging host manually. (2) Remove Chrome Canary if installed. (3) Use `/chrome → Reconnect extension` if tools stop responding mid-session. (4) Full Chrome restart for unrecoverable backoff loops. (5) Use `claude --chrome` from Terminal rather than relying on `chromeEnabled: true` in settings.
