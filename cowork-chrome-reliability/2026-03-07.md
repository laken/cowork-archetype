# CoWork + Claude in Chrome Reliability Briefing — 2026-03-07

## Reliability Signal

**Broadly reported. This is not isolated.** Multiple open GitHub issues, X/Twitter threads, and community reports confirm that the CoWork + Claude in Chrome integration on macOS is unreliable for a significant subset of users. The most visible signal: [Dan Guido's thread](https://x.com/dguido/status/2015675676766916761) calling out the debugging difficulty has become a community reference point. Bug reports span late February through early March 2026, with new issues still being filed as of March 1 ([Issue #29933](https://github.com/anthropics/claude-code/issues/29933)).

Community sentiment: frustrated but not abandoning. Users want this to work and are filing detailed reproduction steps. The tone is "this should be fixable" rather than "this is fundamentally broken."

## Failure Modes Observed

1. **Native messaging host conflict (Cowork vs Claude Code)** — The dominant issue. Both register configs for the same Chrome extension but use incompatible socket locations/formats. Everything appears correct but connection silently fails. ([Issue #29057](https://github.com/anthropics/claude-code/issues/29057), [Dan Guido on X](https://x.com/dguido/status/2015675500799173013))

2. **"No Chrome extension connected" on Apple Silicon** — Persistent failure on macOS Sequoia + M4 Pro. Toggle/restart/reinstall doesn't resolve. ([Issue #29933](https://github.com/anthropics/claude-code/issues/29933))

3. **MCP registry connection failure after auto-update** — Broke around Feb 13-14 coinciding with Claude Desktop v1.1.3189 update. Cowork can't connect to mcp-registry or Claude in Chrome MCP servers. ([Issue #27492](https://github.com/anthropics/claude-code/issues/27492))

4. **Mid-session silent disconnects** — Extension drops connection without warning during use, enters infinite exponential backoff reconnection loop (100+ attempts over ~2.5 hours), never recovers without full Chrome restart. ([Issue #26449](https://github.com/anthropics/claude-code/issues/26449), [Issue #28013](https://github.com/anthropics/claude-code/issues/28013))

5. **Chrome crashes on Apple Silicon** — Chrome fully quits (all windows close) within 2-3 minutes of any browser automation. ([Issue #26305](https://github.com/anthropics/claude-code/issues/26305))

6. **Stale native host wrapper** — `~/.claude/chrome/chrome-native-host` can have a hardcoded version that becomes stale after updates.

## Updates Today

**No new extension or app version released today (March 7).** The most recent notable changes:

- Claude in Chrome is now in beta for all paid plans (Pro, Max, Team, Enterprise), no longer Max-only.
- Claude in Chrome defaults to Haiku 4.5 for speed (Pro plan limitation that users dislike).
- Cowork added scheduled tasks, workflow recording, and a Customize section for plugins/skills/connectors.
- Cowork launched on Windows with full feature parity.
- No specific bug fixes for the native messaging host conflict or Apple Silicon issues have been announced in public release notes.

The [Claude in Chrome Troubleshooting page](https://support.claude.com/en/articles/12902405-claude-in-chrome-troubleshooting) was updated recently but contains the same general advice (restart, update, check permissions). No acknowledgment of the native host conflict issue in official docs.

## Trend (vs. March 6)

**Flat — fourth consecutive day with no fix for core failure modes.** All issues tracked since March 4 remain open. No new extension version, no new Claude Desktop patch, no new guidance in official docs. The release notes page still 404s.

The only movement this week: Claude in Chrome expanded from Max-only to all paid plans, and Haiku 4.5 became the default model (a speed improvement but also a capability downgrade that users are unhappy about). Neither change addresses the reliability bugs.

**Bottom line:** The integration remains stable if you only use Cowork OR Claude Code (not both) and follow the correct startup sequence. The dual-registration native host conflict — the #1 issue — has been known for at least two weeks with no fix announced. Mid-session disconnects and Apple Silicon crashes also persist. Community workarounds (rename config files, single Chrome profile, correct startup order) remain the only path to reliability.
