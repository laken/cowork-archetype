How to read this matrix: Each row represents a data sensitivity tier. Each column represents what the AI agent does with that data. The cell color shows the minimum authorization required before that action should be permitted. Green cells are safe for individual discretion. Red cells require governance infrastructure before the agent touches the data at all.

The diagonal pattern matters: Risk increases along two axes simultaneously — more sensitive data and more consequential actions. The upper-left is where productivity gains live with minimal risk. The lower-right is where firms get in trouble. Most executives start in the upper-left and drift toward the lower-right without noticing the transition.

DPA status shifts cells: A Data Processing Agreement with the AI provider can move "IT Approval" cells to "IT Awareness" and "Formal Review" cells to "IT Approval." Without a DPA, cells shift one level more restrictive. Enterprise/commercial plans with contractual commitments shift one level less.
